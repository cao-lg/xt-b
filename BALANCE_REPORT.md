> 调研前十放置游戏平衡模型 → 搭建全要素模拟系统 → 大量模拟 → 形成新平衡系统 → 执行更新。
> 本次为**纯数值重平衡**，冻结的战斗公式与平衡底线（攻防血固定值加法 + 仅大道比例倍率、境界战力带、命中/闪避/暴击加法、`pillK=0`、突破成本公式、仙缘 sqrt 公式）**一律不改**，仅调整 6 个缩放常量。

## 一、调研结论（放置游戏 top 平衡范式）

- **成本增长**：健康区间 `r ≈ 1.07~1.15`/级；生产 = 线性固定值 × 百分比倍率；倍率区应**隔离相乘**（不同区相乘而非相加），避免互相稀释。
- **攻防设计**：分离倍率区；后期用软封顶或 `dodge/(1+dodge)` 归一化（本项目已采用）；命中/闪避/暴击纯加法不封顶（本项目已采用）。
- **转生(Prestige)**：AdVenture Capitalist 天使 `floor(√(L/K) − √(prevL/K))` 的 sqrt 缩放（翻倍收入需累计 ×4）；Clicker Heroes `(Z−80)/25)^1.3`。多层转生（AD）让每轮收益递减但持续有意义。**本项目仙缘已采用 sqrt 公式**，本次保留并验证其复合行为。
- **ROI**：单位边际 ROI 随等级几何衰减 `×(1/r)`；当某系统边际 ROI 低于另一系统时应切换投入。
- **难度缩放**：Boss 血宜 `×~1.025^zone` 平缓递增；地图敌人属性应随玩家境界增速匹配，否则终局内容不可达。

## 二、模拟系统（tools/balance_sim.js）

加载**真实** `data.js` + `game.js`（经 `new Function` + `localStorage`/`window` shim），用可控 RNG(mulberry32) 做大规模模拟，四大模块：

| 模块 | 内容 | 方法 |
|---|---|---|
| **A 成长/时间** | 到达各境界耗时、破境节奏、飞升转生多轮回加速 | 事件驱动经济循环（买最优 ROI 的功法/洞府/悟道，可破境则破，达真仙则转生） |
| **B 投入产出** | 各系统「买下一级」的边际速度/战力增益、成本、回本时间、ROI | 快照法：直接测买前后 `currentSpeed`/`combatStats` 差值 |
| **C 战斗** | 8 张地图 + 试炼塔在各境界的胜率矩阵 | 蒙特卡洛（每关 100+ 次 `simulateCombat`），用经济到达各境界时的养成快照 + 满级觉醒法宝 + 满喂灵宠 |
| **D 仙缘** | `legacyGain`/`legacyMult` 曲线、每轮回加速比 | 解析 + 轨迹模拟 |

支持 `OVERRIDE="cultBandMult=2.2,realmCostGrowth=3.2"` 环境变量在**内存中覆盖 CONFIG**做假设推演，不改动源文件，便于再调参。

## 三、关键发现（改前）

1. **致命卡死**：原 `realmCostGrowth: 9.0` 使每大境界突破成本 ×9，而速度仅 `×1.5`(修炼带) + 功法洞府线性固定值加法。成本增速远超速度增速 → **元婴之后永久卡死，90 天也到不了化神**，游戏实质上不可通关。
2. **ROI 衰减健康但被卡死掩盖**：早期功法回本仅数分钟；后期(大乘)功法回本 1000+ 天——印证成本/速度失配。
3. **战斗门控基本合理**：可达境界内地图 100% 可胜（地图随玩家成长）；但**终局两张地图（大乘遗迹/天道台）在原战斗带 `1.8` 下 0% 可胜**——敌人属性增速(~2.4×/境界)快于玩家战斗带(1.8×/境界)。
4. **仙缘曲线良好**：sqrt 公式在 `totalXp=5e9` 给 +316 仙缘 → ×10.48；且多轮回复合后**收敛**（每轮回速度增益线性增长 → 单轮回耗时趋近下限），无真正失控。

## 四、新平衡系统（已执行）

仅调整 6 个缩放常量（公式不变）：

| 参数 | 原值 | 新值 | 目的 |
|---|---|---|---|
| `cultBandMult` | 1.5 | **2.2** | 修炼速度随境界 ×2.2，追上突破成本 |
| `realmCostGrowth` | 9.00 | **3.2** | 每大境界突破成本 ×3.2（解除中后期卡死） |
| `majorBreakMult` | 3.2 | **2.5** | 大境界突破额外成本放缓，留余量 |
| `layerCostGrowth` | 2.10 | **2.0** | 层内突破成本更平滑 |
| `cultBandLayer` | 0.04 | **0.05** | 层内速度爬升更顺 |
| `combat.bandMult` | 1.8 | **2.3** | 战斗随境界 ×2.3，匹配敌人增速，令终局地图可达 |

> 仙缘(`legacyPerPoint=0.03`/`legacyGainBase=5e6`)、法宝强化(`enhanceGrowth=1.55`)、觉醒等**全部保留**（已验证健康，无失控）。

## 五、验证结果（改后，真实文件）

- **A 成长**：全部 10 境界在首轮回 100% 可达；逐境界耗时 筑基5h → 金丹12.6h → 元婴23.5h → 化神1.6d → 炼虚2.6d → 合体3.9d → 大乘6d → 渡劫8.9d → **真仙13.2d**（连续在线；含离线 8h/50% 上限则更长，休闲玩家约 1 个月首轮回，旅程感充足）。相邻境界平均耗时比 **1.70×**（渐次变长，节奏满意）。
- **A 多轮回**：飞升转生正确复合加速 —— 第1轮 13.2d(×2.5) → 第2轮 10.2d(×4.6) → 第3轮 5.5d(×7.2) → 第4轮 3.5d(×10.3) → … 最终单轮回收敛到数分钟（终局爽感，非失控）。
- **C 战斗**：每张地图 BOSS 在**解锁境界 78~100% 可胜、下一境界 100% 可胜**（保留挑战门控）；终局 **天道台 渡劫 85%、真仙 100%**；**大乘遗迹 大乘 82%**，均可达。早期地图(妖兽森林/上古战场)维持 100%（战斗带仅影响境界≥1，早期不受改动影响）。
- **回归测试**：逻辑 32/32（含「平衡底线未破坏：breakCost 公式 / combatFormula 7 行」）、战斗 33/33（8 图 BOSS 胜率>50%）、浏览器冒烟 9/9**零报错**（combo/天降机缘/购买/破境/觉醒/切页/大数超界/读档）。
